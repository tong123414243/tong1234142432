类对象

概念

所有类，都存在一个类对象，这个类对象用于提供这个类本身的信息，比如有几种构造方法、有多少属性，多少实例方法。

获取类对象的方式

  1. Class.forName
  2. GaRen.Class
  3. new GaRen().getClass()
    在一个JVM中，一种类只存在一个类对象。上述3种方式取出来的类对象都是一样的。(准确地讲，在一个ClassLoader下，一种类，只会存在一个类对象。而通常在一个JVM下，只会有一个ClassLoader。)

类属性被初始化问题

当获取类对象的时候，会导致类属性被初始化。3种获取类对象方式中，只有 Class pClass2 = GaRen.class 这种方式，不会导致类属性被初始化。

用处

使用反射方式和配置文件。可以通过取出配置文件中的类名称和方法名称，然后通过反射方式去调用这个方法。 这样，在业务方法较多的情况下，可以通过切换调用到第x个业务方法，不需要修改一行代码，也不需要重新编译。
这是Spring框架的最基本原理。

与传统的不同

与传统的通过new 来获取对象的方式不同反射机制，会先拿到Hero的“类对象”,然后通过类对象获取“构造器对象”
再通过构造器对象创建一个对象.

getField与getDeclaredField的区别：
 都用于获取字段。
 但是getField只获取public，包括从父类继承来的字段；
 getDeclaredField 可以获取本类所有字段，但不能获取继承来的字段。（这里只能获取到private的字段，但并不能访问该private字段的值,除非加上setAccessible(true))
